学习stm32

理解**“配置时钟 -> 初始化外设 -> 编写逻辑”**这个流程。

针对 STM32F103C8T6

1. RCC (复位与时钟控制)

总结： RCC 是单片机的“心脏”。STM32 为了省电，默认所有外设的时钟都是关闭的。不开启 RCC 时钟，操作任何外设寄存器都无效。

  • 关键点：F103 的 GPIO 挂在 APB2 总线上。

学习例子：开启 GPIOC 时钟

2. GPIO (通用输入输出)

总结： 它是单片机与外界交流的“触角”。

  • 输出模式：推挽输出（Push-Pull，控制 LED、蜂鸣器）、开漏输出（Open-Drain，用于 I2C）。
  • 输入模式：上拉/下拉输入（检测按键）。
  • C8T6 特点：板载 LED 通常接在 PC13。

学习例子：点亮 PC13 的 LED

3. NVIC & EXTI (外部中断)

总结： 中断让单片机能“停下手头的工作，紧急处理突发事件”。

  • EXTI：外部中断控制器，监测引脚电平变化。
  • NVIC：嵌套向量中断控制器，负责管辖所有中断的“优先级”。

学习例子：按键（PA0）触发中断

  1. 配置 GPIO 为输入。
  2. 配置 AFIO（复用功能时钟），将引脚映射到中断线。
  3. 配置 EXTI（选择触发方式，如下降沿）。
  4. 配置 NVIC（设置优先级）。
  5. 编写中断服务函数（名字必须固定，如 EXTI0_IRQHandler）。

💡 学习路径建议 (C8T6 专用)

  1. 工程模板：先搭建好标准库（StdPeriph）或 HAL 库的开发环境（建议初学者从标准库理解寄存器，从 HAL 库上手项目）。
  2. Debug 技能：学会用 ST-Link 进行硬件仿真。观察变量变化，而不是只靠看 LED 闪烁。
  3. 查阅手册：手边常备《STM32F10x 中文参考手册》。当你疑惑 RCC_APB2Periph_GPIOC 是什么时，去查 RCC_APB2ENR 寄存器。

在 STM32 项目（或任何多文件 C 项目）中，全局变量 ccr（用于在中断和主循环间共享 PWM 占空比值）应该严格遵守“一个定义，多处声明”的原则。

正确做法如下（三种文件怎么写）：

推荐写法（最清晰、最不容易出错）

1. pwm.h （头文件，只做声明）

  • 这里用 extern + volatile 声明。
  • 不要写 =0，也不要写 uint16_t ccr;（那样会变成定义，会导致多重定义错误）。

2. main.c （或其他 .c 文件，只有一个地方放定义）

通常建议把实际定义（带初始值的地方）放在 main.c 里，因为 main 是程序的“主人”。

3. pwm.c （使用变量的地方）

总结：三个文件里分别怎么写 ccr

文件 应该怎么写 说明
pwm.h extern volatile uint16_t ccr; 只声明，不定义，不初始化
main.c volatile uint16_t ccr = 0; 真正的定义 + 初始值（只写一次）
pwm.c 什么都不写（直接用 ccr 即可） 因为包含了 pwm.h，已经知道变量存在

常见错误 & 后果

  • 如果你在 pwm.h 里写了 volatile uint16_t ccr = 0;
    → 每个包含 pwm.h 的 .c 文件都会定义一个 ccr → 多重定义链接错误

  • 如果你在 pwm.c 和 main.c 都写了 volatile uint16_t ccr = 0;
    → 同样多重定义错误

  • 如果三个文件都没写 volatile
    → 中断里改的值，主循环可能看不到最新值（编译器优化问题）

  • 如果忘了 #include "pwm.h"
    → 编译器报 ccr 未声明
