C语言基础笔记

C 语言核心笔记：内存与结构体

1. 字符数组定义的区别

在 C 语言中，初始化字符数组的方式决定了内存布局：

  • 双引号形式：char buf[] = "abc";

  • 特点：编译器会自动在末尾添加 \0。

  • 安全性：高。使用 strlen() 或 printf("%s") 时能正确找到结束标志。

  • 大小：sizeof(buf) 为 4（包含 \0）。

  • 大括号列表形式：char buf[] = {'a', 'b', 'c'};

  • 特点：编译器不会自动添加 \0。

  • 风险：对 strlen() 不友好，容易产生内存越界访问，除非手动写成 {'a', 'b', 'c', '\0'}。

  • 大小：sizeof(buf) 为 3。

对数组名 使用sizeof 返回整个数组的大小
对指针 使用sizeof返回指针变量本身的大小
static关键字对变量和函数的作用：用于局部变量时，变量存放在静态存储区,延长其生命周期至程序结束；用于全局变量或者函数时，将其作用域限制在当前文件内,防止与其他文件的同名变量冲突。
sizeof 是编译期计算内存大小；strlen 是运行期统计字符串长度。

sizeof是计算数据类型或者变量占用的字节数，strlen是计算字符串的有效长度。

A. 进制切换（最常考）

  • %x / %X：打印 16 进制。面试常让你打印一个指针或寄存器值（如 0xAA66）。
  • %o：打印 8 进制。
  • # 标志：会自动加上进制前缀。例如 %#x 会输出 0xaa66，而 %x 只输出 aa66。

B. 宽度与补齐（对齐输出）

  • %5d：输出宽度为 5，空格左补齐。
  • %05d：输出宽度为 5，前面补 0（常用于日期打印，如 2024-05-01）。
  • %-5d：左对齐，右边补空格。

C. 浮点数精度控制

  • %.2f：保留 2 位小数。

  • %e：以 科学计数法 指数形式打印。

2. 结构体字节对齐原则 (Struct Alignment)

为了提高 CPU 访问内存的效率，编译器会进行字节对齐：

  1. 自然对齐：每个成员的起始地址必须是该成员大小（或编译器默认对齐数）的整数倍。
  2. 总体对齐：结构体的总大小，必须是其最大成员大小的整数倍。如果不满足，编译器会在末尾填充（Padding）。

Q3: 关键字 volatile 的作用

问题： 为什么在嵌入式开发中，指向硬件寄存器的指针一定要加 volatile？
答案：

  1. 防止编译器优化：编译器如果发现代码中多次读取同一个地址而没有修改它，可能会为了加速直接从 CPU 寄存器读旧值，而不是从内存/硬件地址读。
  2. 确保实时性：硬件寄存器的值可能随时由硬件（如传感器数据到达）改变，volatile 告诉编译器：每次都要重新去该地址读取最新的值。
  3. 典型场景：状态寄存器查询、中断服务程序（ISR）修改的全局变量、多任务共享的变量。

Q4: 什么是“野指针”？如何避免？

答案：

  • 定义：指向一个已删除的对象或未申请的、非法内存区域的指针。
  • 成因：
  1. 指针定义时未初始化（随机值）。
  2. 指针被 free 或 delete 后没有置为 NULL。
  3. 指针指向了局部变量的地址（函数返回后该地址已失效）。
  • 避免方法：定义指针时立即初始化为 NULL；释放后立即置为 NULL。

格式符 printf 的格式控制符（即 % 后面的字母）

格式符 含义 笔试重点
%d 有符号十进制整数 处理负数时会显示负号。
%u 无符号十进制整数 常用于打印 unsigned int，如果把 -1 用 %u 打印，会得到一个巨大的正数。
%c 单个字符 对应 ASCII 码。注意 char a = '0'; 打印 %d 是 48。
%s 字符串 遇到 \0 停止。常考 “由于没有 \0 导致 printf 越界” 的题目。
%o 八进制整数 较少考，但在某些协议分析题中会出现。

%p (Pointer)：

  • 含义：以十六进制形式输出地址。
  • 特点：会自动加上 0x 前缀（在大多数编译器中），并且会根据系统的位数（32位或64位）补齐长度。
  • 笔试陷阱：面试官常问 %p 和 %x 输出地址的区别。%p 是专门为指针设计的，能保证打印出指针的完整宽度。

%x / %X (Hexadecimal)：

  • 含义：以十六进制输出整数。

  • 特点：%x 输出小写字母（a-f），%X 输出大写字母（A-F）。它不会自动加 0x。

  • 注意：在打印地址时，如果地址高位有 0，%x 往往会把它忽略，而 %p 会保留。

宽度控制 %5d：

  • 输出至少占 5 格，不够就在左边补空格。如果实际数字超过 5 位，按原样输出。

对齐控制 %-5d：

  • 左对齐，空格补在右边。

前导补零 %05d：

  • 如果不满 5 位，前面补 0（常用于显示时间，如 09:30）。

精度控制 %.2f：

  • 保留两位小数（四舍五入）。

长度修饰 %ld / %lld：

  • %ld 用于 long，%lld 用于 long long（64位整数）。如果用 %d 打印 64 位整数，会导致高位截断。

C 语言笔试深度解析：指针运算与内存布局

这道题属于 C 语言笔试中的“深水区”，重点考察指针运算规则、数组名本质以及计算机底层存储（大端/小端）。

A. 指针运算的“步长” (Pointer Arithmetic)

当你对一个指针进行 +1 操作时，系统加的不是数字 1，而是该指针指向的类型所占的大小。

  • p 是结构体指针，结构体大小是 20 字节，所以 p + 0x1 实际地址增加了 20（十六进制为 0x14）。

B. 强制类型转换的“欺骗”

代码中的括号 (unsigned long)p 就像是一个“伪装面具”：

  • (unsigned long)p：你告诉编译器：“别把 p 当指针了，把它当成一个普通的整数！”。既然是普通整数，+1 就是数学意义上的加 1。
  • (unsigned int\*)p：你告诉编译器：“把 p 当成指向 int 的指针”。因为 int 通常占 4 字节，所以 +1 实际地址增加了 4。

C. 十六进制与内存地址

0x 开头的数字是十六进制。在调试内存时，0x1 就是 1，0x14 就是 20。

1. 数组名的“两个例外”—–这俩例外都是指的整个数组

在 C 语言中，数组名 a 通常会“退化”为指向首元素的指针（即 &a[0]）。数组名 a 在不同语境下的含义不同：

  • a (作为地址时): 代表数组首元素的地址，类型是 int *。
  • &a: 代表整个数组的地址，类型是 int (*)[5]（指向包含 5 个整数的数组的指针）。

在以下两种情况下，数组名代表整个数组实体：

  1. sizeof(a)：返回整个数组占用的内存大小（如 $4 \times 4 = 16$ 字节）。
  2. &a：取的是整个数组的地址。虽然数值上和首元素地址相同，但其指针类型不同。

举例对比

  • a + 1：跨越一个 int（4 字节）。
  • &a + 1：跨越整个数组（16 字节）。

本题应用：ptr1 = (int *)(&a + 1)，意味着 ptr1 指向了数组 a 最后一个元素 4 后面紧邻的内存位置。

2. 详解 ptr2：强制类型转换的陷阱

这是全题最绕的地方，注意代码里的强制类型转换：

ptr2 = (int *)((int)a + 1);

  • (int)a：将地址（指针）强行转为一个普通的整数。
    • 假设 a 的地址是 0x2000。
    • 那么 (int)a + 1 的结果就是简单的数学加法：0x2001。
  • (int \*)：再把数值 0x2001 重新变回指针。
  • 结果：ptr2 指向了数组首元素地址往后偏移 1 个字节的地方，而不是往后挪一个 int。

3. 底层原理：小端存储（Little Endian）

为了算出 *ptr2 的值，我们需要观察内存中 1 和 2 的具体分布（假设 int 占 4 字节）：

内存地址 0x2000 (a) 0x2001 (ptr2) 0x2002 0x2003 0x2004 (a+1)
存储内容 01 00 00 00 02
数据归属 元素 1 的低位 元素 1 元素 1 元素 1 的高位 元素 2 的低位
  • 小端模式规则：数据的低位存放在内存的低地址。因此数字 1 存储为 01 00 00 00。
  • ptr2 寻址：执行 *ptr2 时，CPU 从 0x2001 地址开始读取 4 个字节。
  • 读取结果：按顺序取出的字节为 00 00 00 02。
  • 最终数值：在十六进制下，该值拼接回来的结果就是 2000000。

4. 总结与答案

  • ptr1[-1]：由于 ptr1 指向数组末尾之后，[-1] 相当于 *(ptr1 - 1)，即向前挪一个 int 的位置，指向数字 4。输出：4。
  • \*ptr2：由于强转整数加 1 导致指针只移动了 1 字节，发生了“字节错位”。根据小端存储，它跨越了元素 1 和元素 2 的边界。输出：2000000（十六进制）。

  • 第一次：a = 6, b = 6
  • 第二次：a = 7, b = 6 解析： a 是静态局部变量，存放在数据段，函数退出后值保留，下次进入不再执行初始化；b 是自动变量（栈上），函数退出即销毁，每次重新初始化。
  1. extern 是 C/C++ 中用于声明外部链接性的关键字。简单来说，它的作用是告诉编译器：“这个变量或函数已经存在了，但不在当前的这个文件里，请去别的文件找它。”

extern 与 static 的对立

这两个关键字在控制**链接性（Linkage）**上是相反的：

  • static 修饰全局变量： 限制该变量只能在当前文件内访问（内部链接）。
  • extern 修饰全局变量： 允许该变量被工程中其他文件访问（外部链接）。

注意： 如果一个变量被声明为 static，你在另一个文件中使用 extern 也是无法访问到它的，链接时会报错。

“如果在 header.h 中写了 int a = 10;，然后两个 .c 文件都包含了这个头文件，会发生什么？”

  • 回答： 会发生 Linker Error (Multiple Definition)。因为头文件被展开后，两个文件都定义了同一个变量 a。
  • 解决： 应该在头文件中写 extern int a;（声明），并在其中一个 .c 文件中写 int a = 10;（定义）。
5. 结构体的总大小必须是其最大成员类型大小的整数倍。如果不满足，编译器会在末尾填充字节
struct Example {
    char a;    // 1字节
    int b;     // 4字节
    char c;    // 1字节
};

·a 占用第 0 字节。

·b 是 int（4字节），不能从第 1 字节开始，必须跳过 3 个字节，从第 4 字节开始。

·c 占用第 8 字节。

·此时总长 9 字节，但最大成员 b 是 4 字节，总长度必须补齐到 4 的倍数，即 12 字节。

联合体中的所有成员共享同一块内存。在同一时刻，只能存储其中一个成员的值。
联合体的总大小必须是其所有成员类型中最大基本类型大小的整数倍

union Data {
    char a;      // 1字节
    int b;       // 4字节
    double c;    // 8字节
};

·最大成员是 c（8字节）。

·最大基本类型是对齐要求为 8 的 double。

·因此，该联合体的大小为 8 字节。
