单片机汇总知识点

（主要是针对stm32）

1. GPIO

IO有哪三种常用方式？

  1. 轮询（Polling）：CPU不断检测状态寄存器。
  2. 中断（Interrupt）：外设准备好后主动通知CPU。
  3. DMA（Direct Memory Access）：外设与内存直接交换数据，不经过CPU。

状态怎么转换？

通常通过配置控制寄存器（如STM32的MODER, OTYPER, PUPDR）实现转换。

  • 输入（Input）：高阻、上拉或下拉。
  • 输出（Output）：推挽（Push-Pull）或开漏（Open-Drain）。
  • 复用（Alternate Function）：连接到内部外设（如UART, SPI）。
  • 模拟（Analog）：连接到ADC/DAC。

2. 中断 (Interrupt)

什么是中断？

指CPU在执行程序过程中，由于内部或外部事件的触发，暂时中止当前程序，转去处理紧急事件，处理完毕后再返回原处继续执行的过程。

中断的触发方式有哪些？

  • 电平触发：高电平或低电平触发。
  • 边沿触发：上升沿、下降沿或双边沿触发。

哪些函数不能在中断中使用？

  • 包含休眠/阻塞机制的函数（如 delay，malloc，printf）。
  • 获取信号量/互斥锁（可能导致死锁）。
  • 不可重入函数。

什么叫不可重入函数？

指不能在多个任务中同时调用，或者在被中断后再次调用会产生不可预料结果的函数。通常是因为函数内部使用了全局变量、静态变量或标准输入输出。

中断与异常有何区别？

  • 中断：通常指来自CPU外部的异步事件（如按键、定时器）。
  • 异常：通常指来自CPU内部同步产生的事件（如除零、指令非法、内存访问越界）。

中断的响应执行流程是什么？

  1. 中断请求 -> 2. 中断屏蔽位检查 -> 3. 中断优先级判定 -> 4. 保护现场（压栈） -> 5. 执行中断服务函数(ISR) -> 6. 恢复现场（出栈） -> 7. 中断返回。

写一个中断服务需要注意哪些？

  • 快进快出：代码量要小，执行时间要短。
  • 不得包含耗时循环或阻塞操作。
  • 避免返回值和参数传递。
  • 使用 volatile 关键字修饰在ISR中修改的全局变量。

中断和轮询哪个效率高？

  • 效率：通常中断更高，因为CPU不需要无谓地等待。
  • 决策：
    • 数据随机、频率低且实时性要求高：中断。
    • 数据密集且持续（如高速数据流）：轮询或DMA（避免频繁中断导致系统崩溃）。

中断优先级有哪些？

  • 抢占优先级：高抢占优先级可以打断低抢占优先级。
  • 响应优先级（子优先级）：当抢占优先级相同时，决定谁先被处理。

为什么中断函数不能有输入参数和返回值？

中断是由硬件随机触发的，没有调用者（Caller）去显式传递参数，也没有接收者去处理返回值。所有寄存器状态必须在中断返回时恢复原状。

3. 定时器 (Timer)

  • 特殊模式：输入捕获（测频率）、输出比较、PWM模式、单脉冲模式、编码器接口模式。
  • PWM实现调速：改变占空比（Duty Cycle）来改变电机的平均电压。

4. 通信协议

异步传输 vs 同步传输

  • 异步：不需要时钟线，靠双方约定好的波特率同步，有起始位和停止位（如UART）。
  • 同步：有专门的时钟线（CLK），在时钟边沿采样（如SPI, IIC）。

UART、IIC、SPI 对比

协议 线数 模式 速度 距离 优点 缺点
UART 2 异步/全双工 低 中 硬件简单 严格波特率匹配
IIC 2 同步/半双工 中 短 支持多机，线少 协议稍复杂
SPI 3/4 同步/全双工 高 短 速度极快 线多，无应答机制

5. UART 串口

  • USART vs UART：USART支持同步模式，UART仅支持异步模式。
  • 配置参数：波特率、起始位、数据位（8/9）、校验位（None/Odd/Even）、停止位。
  • RX上拉原因：确保空闲时电平稳定，防止漂移产生乱码。
  • 传输浮点数：
    1. 使用共用体 union 将浮点数转为4字节数组发送。
    2. 使用 sprintf 转成字符串发送。
  • 接收不定长数据：
    1. 串口中断+空闲中断（IDLE） + DMA（STM32常用）。
    2. 设置超时时间。

6. IIC (I2C)

  • 信号：起始信号（SCL高时SDA下降沿）、停止信号（SCL高时SDA上升沿）、ACK/NACK信号。

  • 上拉原因：IIC是开漏输出，必须通过外部上拉电阻提供高电平。

  • 仲裁机制：通过“线与”逻辑，发送高电平但检测到总线为低电平的设备失去控制权。

  • 1.8V与3.3V通信（无转换芯片）：

    利用IIC开漏特性，将上拉电阻连接至1.8V电源。只要3.3V主机的IO口能识别1.8V的高电平阈值（$V_{IH}$），即可安全通信。

7. SPI

  • 速度影响因素：时钟线电容负载、主从机频率限制、PCB布线阻抗。
  • 四种模式：由 CPOL（时钟极性）和 CPHA（时钟相位）决定。

8. DMA & ADC

DMA

  • 机制：由专用控制器申请总线控制权，在存储器和外设间直接搬运数据，无需CPU参与，搬运完成后通过中断通知CPU。

ADC

  • 实现方式：逐次逼近型（SAR）、积分型、Σ-Δ型。
  • 滤波方法：均值滤波、中值滤波、一阶递推滤波、加权平均滤波。

9. 存储与启动

NAND vs NOR Flash

  • NOR：支持XIP（片上执行），读取快，容量小，可靠性高。
  • NAND：以块/页为单位读写，容量大，成本低，可能有坏块。

MCU 启动流程

  1. 上电复位。
  2. 从启动配置处（Flash/System Memory）加载堆栈顶地址。
  3. 加载复位向量（Reset Handler）。
  4. 初始化时钟、栈、堆。
  5. 跳转至 main。

10. ARM 体系结构

  • 寄存器：R0~R15。R13(SP), R14(LR), R15(PC)。
  • CPSR：当前程序状态寄存器。SPSR：程序状态保存寄存器（异常切换时保存CPSR）。
  • 哈佛结构：指令和数据总线分开。冯诺依曼：指令和数据共用总线。
  • 参数传递：前4个参数通过 R0-R3 传递，超过的压栈。

11. 其他问题

  • 交叉编译：在A平台（如PC）上编译出能在B平台（如ARM）上运行的二进制程序。因为单片机性能不足以运行编译器。

  • 0x100000 赋值：

    C

  • 51单片机PSW：Program Status Word（程序状态字），包含CY（进位）、AC（辅助进位）、OV（溢出）等标志。
