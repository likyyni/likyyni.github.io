前端学习

1. 运算符 (Operators)

  • 优先级：括号 > 单目 > 算术 > 关系 > 逻辑 > 三目 > 赋值 > 逗号。
  • 自增自减：
    • a++ = b++ 错误：a++ 返回的是一个临时值（右值），不能被赋值。
    • ++a = ++b 正确（在 C++ 中）：++a 返回的是变量本身的引用（左值）。但在标准 C 中，两者通常都被视为右值，不可赋值。
  • 效率：++i 比 i++ 效率高。因为 i++ 需要产生一个临时变量来保存旧值，而 ++i 直接对变量操作并返回。

2. 输入输出 (I/O)

  • scanf vs gets：scanf 遇到空格、制表符、回车会停止；gets 只在遇到回车时停止（但 gets 不安全，易导致溢出，建议用 fgets）。
  • 抑制符 \*：scanf("%*d %d", &a); 表示跳过第一个整数，将第二个整数存入 a。
  • printf 压栈：printf 的参数是从右向左压栈计算的，所以 printf("%d %d", *ptr, *(++ptr)) 的结果取决于编译器对表达式求值的顺序（通常先算右边）。

3. 预处理 (Pre-processing)

  • 编译过程：预处理 -> 编译 -> 汇编 -> 链接。
  • 宏处理阶段：在预处理阶段进行简单的文本替换。
  • #error：强制编译器停止编译并输出错误信息，常用于检查宏定义或环境配置。
  • 1年秒数：#define SECONDS_IN_YEAR (60UL * 60UL * 24UL * 365UL)（使用 UL 防止溢出）。

4. 关键字 (Keywords)

  • static：
    1. 修饰局部变量：延长生命周期至程序结束，只初始化一次。
    2. 修饰全局变量/函数：限制作用域，使其仅在当前 .c 文件可见。
  • const：
    • const int *p; 指向常量的指针（内容不可变）。
    • int * const p; 指针常量（地址不可变）。
  • volatile：告诉编译器该变量易变，禁止优化（每次必须从内存读取，而非寄存器缓存）。场景：硬件寄存器、中断里的全局变量、多线程共享变量。
  • extern “C”：告诉 C++ 编译器按照 C 语言的规则翻译函数名，解决 C++ 函数重载导致的符号修饰问题。
  • sizeof vs strlen：sizeof 是运算符，编译时确定，返回内存空间大小；strlen 是函数，运行时确定，计算字符串 \0 前的长度。

5. 堆栈 (Heap & Stack)

特性 栈 (Stack) 堆 (Heap)
管理方式 编译器自动分配释放 程序员手动申请释放
空间连续性 连续空间（向低地址生长） 不连续（链表管理）
效率 极高 相对较低
大小 固定且较小（通常几M） 很大，受限于物理内存
  • 溢出原因：栈溢出（无限递归、局部大数组）；堆溢出（内存泄漏、申请空间过大）。

6. 指针 (Pointers)

  • 指针数组 vs 数组指针：
    • int *p[10]; 指针数组：一个数组，里面存了10个指针。
    • int (*p)[10]; 数组指针：一个指针，指向一个含有10个整数的数组。
  • 函数指针：int (*p)(int); 指向函数的地址。常用于回调函数。
  • 野指针：指向已释放或非法内存的指针。避免方法：初始化为 NULL，释放后立即置 NULL。

7. 内存分配与程序结构

  • 内存分区：栈区、堆区、全局/静态存储区、常量存储区、代码区。
  • 内存泄漏：malloc 申请的内存没有 free。
  • malloc(1.2G)：在 32 位系统中，由于用户空间通常只有 3G 且内存碎片存在，很难申请到连续的 1.2G；在 64 位系统且开启虚拟内存时，通常可以成功。

8. 大小端 (Endianness)

  • 大端 (Big-endian)：高位字节存放在低地址。

  • 小端 (Little-endian)：低位字节存放在低地址（STM32/x86 常用）。

9. 核心定义练习

  • int a; (一个整型数)
  • int *a; (一个指向整型数的指针)
  • int **a; (一个指向指针的指针)
  • int a[10]; (一个有10个整型数的数组)
  • int *a[10]; (一个有10个指针的数组)
  • int (*a)[10]; (一个指向有10个整型数数组的指针)
  • int (*a)(int); (一个指向函数的指针)
  • int (*a[10])(int); (一个有10个指针的数组，指向返回整型的函数)

10. 其他高频问题

  • 不用新变量交换 a, b：
    • a = a ^ b; b = a ^ b; a = a ^ b; (异或法，最推荐)
    • a = a + b; b = a - b; a = a - b; (加减法，可能溢出)
  • 计算二进制中1的个数：使用 x = x & (x - 1)。该操作每次会消除二进制中最右边的一个 1。
  • 字节对齐：为了提高 CPU 访问内存的效率。如果不地址对齐，CPU 可能需要两次内存访问才能读完一个变量。
  • 数组退化：当数组作为函数参数传递时，它会退化为指向首元素的指针。所以 void func(int arr[]) 里的 sizeof(arr) 在 64 位系统下结果是 8。
